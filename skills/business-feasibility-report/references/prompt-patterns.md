# 可复用提示词模式

Use these patterns internally to structure analysis. Adapt them to the user's industry and available materials.

## 1. Intake

```text
请先识别该项目的可研类型、目标读者、决策阶段和资料完整度。输出：
1. 项目一句话定义
2. 本报告要支持的关键决策
3. 已有资料
4. 关键缺口
5. 本次分析边界
```

## 2. 用户与市场洞察

```text
针对【项目/产品】，请从真实行为而非人口标签出发，识别3-4类潜在用户或客户。每类包含：
- 具象命名
- 行为特征
- 使用或消费场景
- 支付能力
- 复购/留存可能
- 获客路径
- 是否适合作为核心用户

然后对核心用户拆解痛点、痒点、爽点，并转化为P0/P1/P2商业机会。
```

## 3. 区位与周边配套

```text
针对【项目地址/片区】，输出区位与周边配套研判：
1. 片区属性
2. 周边高校/社区/办公/商业/文旅/公共服务/停车设施统计
3. 每类设施可导入的客群
4. 对项目定位、招商、业态和运营的影响
5. 需现场复核事项

用表格输出：设施名称、类型、距离或时间、可导入客群、对项目影响、需复核事项。
```

## 4. 交通便利性

```text
请统计并研判项目交通便利性：
- 最近地铁站、公交站、主干道、快速路
- 步行、非机动车、网约车、停车、后勤装卸条件
- 对工作日、周末、夜间、活动高峰的影响
- 哪些业态受交通利好，哪些业态受停车或动线限制

区分公开地图初判和需导航/现场复核的数据。
```

## 5. 地图可视化

```text
请为该项目设计一张区位与交通可视化图。要求：
1. 标注项目位置
2. 标注周边配套设施类别
3. 标注地铁、公交、道路、停车、落客点
4. 用不同类别或图例区分
5. 图下说明数据来源、测距口径、复核边界

如无法制作真实地图，输出方位示意图或距离圈层图，并说明其仅用于初步判断。
```

## 6. 硬性条件研判

```text
请从落地条件角度审查该项目，按已确认、初步判断、需复核分类。检查：
权属用途、消防、人流、结构荷载、电力、给排水、排烟排污、停车交通、网络弱电、安防、证照、运营团队。
输出每项对项目方向的影响和前置动作。
```

## 7. 方案比选

```text
请提出2-4个可行方向，并从市场吸引力、资源匹配、投入强度、落地难度、合规风险、收益确定性、分期验证可能性进行矩阵比较。
最后给出推荐路径、备选路径和暂缓路径。
```

## 8. 商业模式

```text
请构建商业模式和运营模型：
- 收入来源
- 成本结构
- 关键资源
- 合作伙伴
- 用户获取
- 运营机制
- 关键指标
- 30/90/180天验证方式

若缺少财务数据，请用假设口径和区间表达，不给伪精确收益。
```

## 9. 风险与决策

```text
请输出风险清单和最终决策建议：
1. 风险、等级、影响、触发条件、控制措施
2. 可立即推进事项
3. 谨慎推进事项
4. 暂缓推进事项
5. 需补充资料清单
6. 下一步3-5个动作
```
