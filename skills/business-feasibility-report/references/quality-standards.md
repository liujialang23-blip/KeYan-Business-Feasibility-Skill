# 商用品质标准

## Report Voice

Use a professional, restrained, decision-oriented voice. The report should sound like a sober feasibility memo, not a sales deck.

Good:

- “在现有资料口径下，可初步判断...”
- “该方向具备验证价值，但不宜作为一期重投入基础。”
- “需完成消防、电力、停车和证照复核后，方可进入施工图或招商承诺。”

Avoid:

- “前景无限”
- “一定盈利”
- “交通极其便利” without statistics
- “政策支持力度巨大” without source
- Precise ROI without assumptions

## Required Decision Standard

Every full report must answer:

- Should the user proceed?
- What should be done first?
- What should be tested before investment?
- What should be avoided?
- What missing information could reverse the conclusion?

## Evidence Language

| Situation | Use |
|---|---|
| Source-backed fact | “根据...资料/公开信息...” |
| Analytical inference | “可初步判断...” |
| Important uncertainty | “需复核...” |
| High-risk assumption | “不宜作为投资依据...” |
| Staged recommendation | “建议先小范围验证...” |

## Location And Map Quality

For physical projects:

- Include at least one map, schematic, or relationship diagram.
- Include a table of surrounding facilities.
- Include a table of transportation conditions.
- Explain how location affects客流,招商,业态,运营,停车,安全.
- State that public map distance is an initial estimate when exact navigation or site measurement is unavailable.

## Table Discipline

Use tables for:

- functional ratios;
- option comparison;
- surrounding facilities;
- traffic nodes;
- hard-condition checks;
- revenue and cost structure;
- risk list;
- staged roadmap;
- supplementary materials.

Tables should have decision-useful columns. Avoid tables that only repeat prose.

## Common Failures

| Failure | Fix |
|---|---|
| Only market concept, no landing condition | Add hard-condition and resource checks |
| Only nearby resources, no conversion logic | Explain which客群 can actually be imported |
| No map for a location project | Add map, point map, ring map, or schematic |
| Treats policy subsidy as guaranteed income | Move policy funds to upside, not base case |
| Gives one recommendation only | Add option comparison and delayed paths |
| Ignores permits and safety | Add compliance and risk table |
| Overuses exact numbers | Add assumptions, source, and sensitivity |

## Final QA Checklist

Before final delivery:

- The first section gives a clear综合结论.
- Evidence and assumptions are separated.
- Location-dependent projects include地图/示意图,周边配套统计,交通便利统计.
- There is at least one option comparison when strategic direction is uncertain.
- Business model includes revenue, cost, resources, operating mechanism, and metrics.
- Risks include level, impact, trigger, and controls.
- Recommendations are split into可立即推进,谨慎推进,暂缓推进.
- Missing materials are listed.
- No claim promises profit, approval, subsidy, traffic, or compliance without evidence.
